高风险

1. AI API Key 明文存库

位置: backend/vocab_qc/core/models/prompt.py:30
ai_api_key: Mapped[Optional[str]] = mapped_column(String(200), nullable=True)
问题: Prompt 表直接以 String(200) 明文持久化 API Key，DB 备份、只读副本、误配置的 SELECT 权限均可无阻读取
表现: 数据库泄露 = API Key 泄露；攻击者可用该 Key 无限调用 AI 服务并产生费用
修复: 应用层加密（Fernet/AES-GCM）写入、解密读取；或仅存环境变量名（AI_KEY_REF），运行时从环境变量取值

—
2. Nginx 仅监听 HTTP 80，无 TLS

位置: deploy/nginx.conf:2
server {
listen 80; # ← 无 443，无 TLS
...
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" # ← HSTS 头已加但无意义
问题: HSTS 响应头已写入（main.py:65），却没有 TLS 端口——HSTS 仅在 HTTPS 连接下生效，现状等于自相矛盾
表现: JWT token、登录密码全程明文；内网或咖啡厅 Wi-Fi 下中间人 1 分钟内可截获任意 session
修复: 配置 TLS（Let's Encrypt / 自签），加 listen 443 ssl，增 80→443 永久重定向；删除 HTTP 下的 HSTS 头

—
3. HTML 过滤正则可绕过 → 存储型 XSS → Token 窃取

位置: backend/vocab_qc/api/routers/review.py:27-39
_HTML_TAG_RE = re.compile(r"<\s*/?[a-zA-Z][^>]*>") # ← 不匹配换行符
_EVENT_HANDLER_RE = re.compile(r"\bon\w+\s*=", re.IGNORECASE)

def _reject_html(text, field_name="content"):
if _HTML_TAG_RE.search(text) or _EVENT_HANDLER_RE.search(text):
raise HTTPException(...) # ← 可绕过
问题: re.compile 默认不跨行（. 不匹配 \n），<img\nsrc=x\nonerror=alert(1)> 可绕过两个正则
表现: 审核员提交绕过 payload → 存入 content_items → 其他用户打开审核页面触发 XSS → 结合高风险 JWT 存 localStorage（中风险 #1），攻击链完整：XSS → localStorage.getItem('auth')
→ 完整 JWT
修复: 用 bleach.clean(text, tags=[], strip=True) 或 html.escape() 替代正则黑名单；入口统一，覆盖 words.py:150 处的 re.sub 同类问题

—
4. SSRF 防护存在 TOCTOU 竞争窗口（DNS Rebinding）

位置: backend/vocab_qc/core/security.py:27
resolved_ip = socket.gethostbyname(hostname) # ← 验证时解析一次
ip = ipaddress.ip_address(resolved_ip)
if ip.is_private or ip.is_loopback ...:
raise ValueError(...)

← httpx 发请求时 DNS 可能再次解析，指向不同 IP
问题: Check 与 Use 之间存在时间窗口；DNS rebinding 攻击者令同一域名先解析为公网 IP（通过验证），再切换到 127.0.0.1（实际请求打内网）
表现: admin 新建 Prompt 填写攻击者控制域名 → 验证通过 → 请求打 http://127.0.0.1:5432（Postgres）或 http://169.254.169.254（云元数据）→ 内网探测
修复: 在 httpx.AsyncClient 层拦截实际连接的对端 IP（transport hook 或自定义 resolver）；或维护已验证域名 TTL 缓存并绑定 IP 发请求

综合攻击链（最高优先级）

高风险 #3 + 中风险 #1 构成完整账号劫持链：

绕过正则 HTML 过滤 → 存储型 XSS 写入 DB
→ 其他用户打开审核页触发
→ JS 读取 localStorage.getItem('auth')
→ 获得完整 JWT (4h 有效期)
→ 以受害者身份调用所有 API（含 export/admin）

建议优先同步修复这两项。
