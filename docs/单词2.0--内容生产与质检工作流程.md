# 英语词汇学习系统 V2.0 — 内容生产与质检工作逻辑文档

## 1. 文档概述

### 1.1 目的

定义英语词汇学习系统 V2.0 内容生产与质检的完整工作流程，覆盖从 Prompt 编写到质检通过的全链路。本文档是可复用的方法论资产，适用于当前词汇内容生产，也适用于未来新增内容维度。

### 1.2 范围

| 包含 | 不包含 |
|------|--------|
| Prompt 编写与版本管理 | 词表导入与数据清洗 |
| 模型选择与评估 | 最终导出与交付前端 |
| 定红线与 Prompt 优化 | 前端平台搭建 |
| 小批量验证与量产 | |
| 质检全流程（算法 + AI + 人工） | |

### 1.3 设计原则

1. **标准先行**：学科标准是唯一仲裁，所有 Prompt 和质检规则从标准派生
2. **原子化可评估**：每条 Prompt 要求拆成单条可量化的检验点
3. **可复用**：整套流程是方法论资产，换内容 / 换模型都能复用
4. **Fail-safe**：未通过质检的内容不会进入下一环节

## 3. Phase 1: Prompt 编写与版本管理

### 3.1 背靠背写作机制

**目的**：防止单人视角导致的逻辑漏洞。

**流程**：

1. **学科标准输入**：学科专家提供每个内容维度的学科标准（如：例句字数限制、语块包含目标词等）
2. **独立编写**：两人各自独立编写同一维度的 Prompt
3. **对比评审**：
   - 逐条对比两版 Prompt 的差异
   - 识别各自遗漏的约束条件
   - 合并最优表述，产出合并版
4. **审核确认**：最终版本由项目负责人确认

### 3.2 Prompt 原子化标准

每条 Prompt 中的要求必须拆解为**可独立评估的原子规则**。以下为基于五维度学科标准拆解的原子规则清单。

> **"通过标准"与"红线"的关系**：下表中的"通过标准"是各规则的**理想目标值**。Phase 3 定红线时，红线可根据实际情况适当调整（上调或下调），但不得偏离通过标准超过 10 个百分点。例如：E2 通过标准 ≥ 90%，则红线可设为 80%–100%。
>
> **硬性规则与可调规则**：通过标准为 100% 的规则（无论评估方式）为**硬性规则**，红线不允许下调，必须保持 100%。通过标准低于 100% 的规则（如 C3 ≥ 90%、E2 ≥ 90%、N6 ≥ 85% 等）为**可调规则**，红线可在通过标准 ±10 个百分点内调整。

#### 维度一：Meaning 语义

| # | 原子规则 | 评估方式 | 通过标准 |
|---|---------|---------|---------|
| M1 | 释义来源基于权威词典（牛津高阶第10版），严禁 AI 模糊生成 | 人工核验 | 100% |
| M2 | 仅保留核心义项（中高考高频考点义项），过滤非核心/罕见/陈旧义项 | 人工判断 | 100% |
| M3 | 词性标签使用统一缩写：n., v., adj., adv., prep., conj., pron., num., art., int. | 自动校验 | 100% |
| M4 | 不同词性之间换行分隔 | 自动校验 | 100% |
| M5 | 同词性下不同义项使用分号 `;` 分隔 | 自动校验 | 100% |
| M6 | 禁止使用括号做解释说明（特殊标注除外） | 自动校验 | 100% |
| M7 | 词义与词性严格匹配 | 人工/AI 判断 | 100% |

#### 维度二：Phonetics 语音

| # | 原子规则 | 评估方式 | 通过标准 |
|---|---------|---------|---------|
| P1 | 统一采用 IPA 国际音标，美式发音（GA） | 自动格式校验 | 100% |
| P2 | 音标符号与音节切分块逻辑一一对应，使用 `·` 分隔（如 pa·per → /ˈpeɪ·pər/） | 自动校验 | 100% |
| P3 | 音标来源为权威词典（牛津/朗文，待定） | 人工核验 | 100% |

#### 维度三：Syllables 音节

| # | 原子规则 | 评估方式 | 通过标准 |
|---|---------|---------|---------|
| S1 | 每个切分块包含且仅包含一个实际发声的元音（词尾成节音 -le 除外） | 半自动校验 | 100% |
| S2 | 统一使用中圆点 `·` 作为分隔符 | 自动校验 | 100% |
| S3 | 二合字母、元音队、R 音节等原子单位禁止从中拆分（如 sh, ee, ph, ar） | 半自动校验 | 100% |
| S4 | 单音节词不做切分 | 自动校验 | 100% |

> **音节的生产方式**：音节切分有独立的 Prompt，走完整的 Phase 1–5 流程（背靠背编写 → 模型评估 → 定红线优化 → 小批量验证 → 量产）。音节 Prompt 在版本管理中独立于语音（Phonetics）维度。

#### 维度四：Chunk 语块 + Sentence 例句

| # | 原子规则 | 评估方式 | 通过标准 |
|---|---------|---------|---------|
| C1 | 语块包含目标词 | 自动匹配 | 100% |
| C2 | 语块长度 2–5 词 | 自动计数 | 100% |
| C3 | 语块为高频固定搭配（动+宾、形+名、动+介等），严禁随意组合 | 人工/AI 判断 | ≥ 90% |
| C4 | 禁止使用括号 | 自动校验 | 100% |
| C5 | 中文对照独立成行，不得包裹在英文后的括号内 | 自动校验 | 100% |
| E1 | 例句语法以初中英语教学大纲为上限 | AI/人工判断 | ≥ 95% |
| E2 | 主干结构优先"主+谓+宾"或"主+系+表"简单陈述句 | AI/人工判断 | ≥ 90% |
| E3 | 仅允许 and/but/so/because 连接的单层逻辑并列句或状语从句 | AI/人工判断 | ≥ 95% |
| E4 | **定语从句**仅允许 that/which/who 引导的限制性定语从句，从句长度 ≤ 5 词（E3 所允许的 because/so 等连接的状语从句不受此限制） | AI/人工判断 | 100% |
| E5 | 严禁"非谓语动词短语作状语"、虚拟语气、倒装句、独立主格 | AI/人工判断 | 100% |
| E6 | 例句必须包含目标词（或其常见变形） | 自动匹配 | 100% |
| E7 | 例句长度 5–20 词 | 自动计数 | 100% |
| E8 | 例句必须配有中文翻译，且中文翻译必须与英文例句语义对应 | 自动校验（非空）+ AI 判断（语义对应） | 100% |
| E9 | 例句必须体现标注的义项，严禁张冠李戴 | AI/人工判断 | 100% |
| E10 | 例句语言必须自然地道，严禁"中式英语" | AI/人工判断 | 100% |
| E11 | 例句内容必须阳光中性，严禁涉及暴力、低俗、性别种族歧视或宗教政治倾向 | AI/人工判断 | 100% |

#### 维度五：Mnemonic 助记

| # | 原子规则 | 评估方式 | 通过标准 |
|---|---------|---------|---------|
| N1 | 助记类型必须从 4 类中选择：词根词缀、词中词、音义联想、考试应用 | 自动校验 | 100% |
| N2 | 每组助记包含完整 4 部分结构：[助记类型] + [核心公式] + [助记口诀/逻辑] + [老师话术] | 自动结构校验 | 100% |
| N3 | 核心公式使用 `+` 或 `≈` 进行逻辑拆解 | 自动校验 | 100% |
| N4 | 口诀/逻辑字数 ≤ 15 字（考试应用类 ≤ 30 字） | 自动计数 | 100% |
| N5 | 老师话术约 500 字，包含完整步骤框架 | 自动计数 + 人工判断 | ≥ 90% |
| N6 | 逻辑合理、不牵强，严禁"伪助记" | 人工判断 | ≥ 85% |
| N7 | 无法合理拆解的词允许该项留空，严禁硬编 | 人工判断 | 100% |

#### 原子化的意义

- **评估方式 = 自动 / 半自动** 的规则 → 算法校验层（Phase 6 Layer 1，见 7.2）
- **评估方式含 AI 判断** 的规则 → AI 语义校验层（Phase 6 Layer 2，见 7.3）
- **评估方式含人工判断** 的规则 → 人工审核 checklist（Phase 7，见 7.4）
- 部分规则的评估涉及多种方式（如 N5、M7、C3），会同时出现在多个质检层
- 每条规则有明确的通过率统计 → 定量评估模型和 Prompt 的效果

### 3.3 版本管理规范

每个维度的 Prompt 需建立版本管理表：

| 字段 | 说明 |
|------|------|
| 版本号 | v1.0, v1.1, v2.0（主版本号 = 重大逻辑变更，次版本号 = 措辞微调） |
| 维度 | meaning / phonetic / syllable / chunk / sentence / mnemonic（注：chunk 和 sentence 概念上同属"语境"维度，但因各自有独立 Prompt，版本管理时分开记录） |
| 变更说明 | 本次修改了什么、为什么改 |
| 变更日期 | YYYY-MM-DD |
| 对应模型 | 该版本在哪个模型上测试过 |
| 测试通过率 | 黄金测试集上的通过率 |
| 状态 | draft → testing → approved → deprecated（见下方说明） |

**状态流转规则**：

- `draft`：新建或修改中的版本
- `testing`：已提交到 Phase 3 迭代流程中、正在用黄金测试集评估的版本
- `approved`：通过红线验证、可用于量产的版本
- `deprecated`：被新版本取代的历史版本

**版本管理规则**：

- 每次修改必须新建版本，不允许覆盖历史版本
- 仅 approved 状态的 Prompt 才能用于量产
- 同一时间每个维度只能有一个 approved 版本

## 4. Phase 2: 模型选择与评估

### 4.1 黄金测试集

| 属性 | 说明 |
|------|------|
| 规模 | 50 个单词 |
| 来源 | 学科组筛选 |
| 覆盖要求 | 覆盖不同年级、不同词性、不同难度层级、多义词 |
| 标准答案 | 每个词的 5 个维度内容均由学科专家人工编写或审核确认 |

### 4.2 多模型对比流程

**候选模型**：Gemini、ChatGPT、豆包

```
同一 Prompt（Phase 1 初稿版本）
         │
    ┌────┼────┐
    ↓    ↓    ↓
 Gemini GPT  豆包
    ↓    ↓    ↓
  各跑 50 词黄金测试集
    ↓    ↓    ↓
    └────┼────┘
         ↓
  按原子规则逐项打分
         ↓
  汇总通过率，选最优模型
```

### 4.3 评估维度

| 评估维度 | 说明 |
|---------|------|
| 各原子规则通过率 | 逐条对比，找出每个模型的强项和短板 |
| 总体通过率 | 所有原子规则的综合通过率 |
| 输出稳定性 | 同一输入多次运行，结果是否一致 |
| 成本 | API 单价 × 调用次数 |

### 4.4 选择决策

- 选择总体通过率最高的模型
- 通过率相近时，优先选择成本更低的
- 记录选择依据，归档到版本管理表

## 5. Phase 3: 定红线与 Prompt 优化

### 5.1 定红线

在优化 Prompt 之前，必须先定义各维度的合格率红线，作为 Prompt 迭代的**目标基准**。

**红线设定逻辑**：

- **硬性规则**（通过标准 = 100%）：红线固定为 100%，无需讨论
- **可调规则**（通过标准 < 100%）：红线可根据实际情况在通过标准 ±10 个百分点内上调或下调
- 红线数值需项目负责人确认后方可生效

> ⚠️ 具体数值待定。红线确定后记录到版本管理表，与 Prompt 版本绑定。

### 5.2 Prompt 优化迭代

**目标**：在选定模型上，通过反复调整 Prompt，使黄金测试集的通过率达到各维度红线。

**迭代流程**：

```
当前 Prompt 版本（draft）
    │
    ↓
用黄金测试集（50词）跑一轮
    │
    ↓
按原子规则逐项统计通过率
    │
    ├── 全部维度达标 → Prompt 状态改为 approved → 进入 Phase 4
    │
    └── 未达标 →
            │  分析失败集中在哪些规则
            │  针对性修改 Prompt（新建版本）
            │  记录变更说明到版本管理表
            └── 回到顶部，重新跑黄金测试集
```

**关键约束**：

- 每次修改 Prompt 必须新建版本（禁止覆盖）
- 每轮迭代结果记录到版本管理表（Prompt 版本 ↔ 通过率）
- 优化过程中如发现模型本身能力不足，可回退到 Phase 2 重新选型

## 6. Phase 4–5: 小批量验证与量产

### 6.1 小批量验证

| 属性 | 说明 |
|------|------|
| 批量规模 | 500 条 |
| 使用模型 | Phase 2 选定的模型 |
| 使用 Prompt | Phase 3 的 approved 版本 |
| 目的 | 确认 Prompt 从 50 词扩大到 500 条后，通过率仍达红线 |

**执行流程**：

1. 用选定模型 + approved Prompt 跑 500 条
2. 全量过算法校验（Layer 1，自动）
3. 人工评估（全量或抽样，视资源而定）
4. 统计各维度通过率和总体通过率
5. 对比红线：
   - 达标 → 进入量产
   - 未达标 → 分析原因，回退 Phase 3 继续优化 Prompt

> Phase 4 的验证范围为 **Layer 1 算法校验 + 人工评估**，不含 Layer 2 AI 语义校验（质检 Prompt 可能仍在 Q1–Q3 并行生产中）。若质检 Prompt 已进入 Q3 阶段，建议对 Phase 4 的一个子集（如 50–100 条）试跑 Layer 2 AI 语义校验，提前验证质检可行性，避免量产后大面积返工。

### 6.2 量产准入 Checklist

满足以下全部条件后方可进入量产：

- [ ] 内容生产 Prompt 状态为 approved
- [ ] 模型已选定并记录
- [ ] 红线数值已确认
- [ ] 小批量通过率达到红线
- [ ] 质检 Prompt 已 approved（Phase Q1–Q3 完成，见 7.3.2 节）

> 质检 Prompt 的生产流程（Q1–Q3）可与 Phase 4 小批量验证并行推进，但必须在量产（Phase 5）启动前完成，否则 Phase 6 质检无法执行。

### 6.3 量产执行规则

- 批量规模：1,000+ 条
- 分批执行，每批记录：批次号、时间、模型版本、Prompt 版本
- 每批独立统计通过率
- 通过率低于红线的批次 → 排查原因后决定是否重跑

## 7. Phase 6–7: 质检流程

### 7.1 质检总览

```
生成内容
    │
    ↓
┌──────────────────┐
│ Layer 1: 算法校验 │  纯规则，准确率 ≈ 100%
│ 格式/字数/包含关系 │  仅标记通过/失败，不自动重试
└────────┬─────────┘
    ┌────┴────┐
    ↓ 通过    ↓ 失败（跳过 Layer 2）
┌──────────┐  │
│Layer 2:  │  │
│AI 语义   │  │
│仅标记    │  │
└────┬─────┘  │
     └────┬───┘
          ↓
┌──────────────────────┐
│ Layer 3: 人工审核      │  最终决策
│ 必审失败项 + 抽检通过项 │
│ 可触发"重新生成"(≤3次) │
└────────┬─────────────┘
         ↓ 通过
      质检通过 ✓
```

### 7.2 Layer 1: 算法校验

**特点**：纯规则匹配，无模型参与，准确率接近 100%。

校验项直接从 3.2 节原子化标准中"评估方式 = 自动 / 半自动"的规则派生：

| 对应规则 | 检查项 | 校验逻辑 | 失败处理 |
|---------|--------|---------|---------|
| M3 | 词性标签格式 | 是否为 n./v./adj. 等标准缩写 | 标记失败 |
| M4 | 词性换行分隔 | 不同词性之间是否换行 | 标记失败 |
| M5 | 义项分号分隔 | 同词性义项是否用 `;` 分隔 | 标记失败 |
| M6 | 禁止括号 | 释义中是否包含 `()` | 标记失败 |
| P1 | 音标格式 | 是否为合法 IPA(GA) 格式 | 标记失败 |
| P2 | 音标-音节对齐 | `·` 分隔数是否与音节一致 | 标记失败 |
| S1 | 元音锚点校验 | 每个切分块是否包含且仅包含一个实际发声元音（-le 除外） | 标记失败 |
| S2 | 音节分隔符 | 是否使用中圆点 `·` | 标记失败 |
| S3 | 原子单位完整性 | 二合字母/元音队/R音节等是否被错误拆分（基于已知原子单位列表） | 标记失败 |
| S4 | 单音节不切分 | 单音节词是否包含分隔符 | 标记失败 |
| C1 | 语块含目标词 | 字符串匹配 | 标记失败 |
| C2 | 语块长度 | 词数是否在 2–5 范围内 | 标记失败 |
| C4/C5 | 语块/例句格式 | 是否包含括号、中文对照是否独立成行 | 标记失败 |
| E6 | 例句含目标词 | 字符串匹配（含常见变形） | 标记失败 |
| E7 | 例句长度 | 词数是否在 5–20 范围内 | 标记失败 |
| E8 | 例句有中文翻译 | 中文翻译字段非空 | 标记失败 |
| N1 | 助记类型 | 是否为 4 类之一 | 标记失败 |
| N2 | 助记结构完整性 | 是否包含 4 部分结构 | 标记失败 |
| N3 | 核心公式符号 | 是否使用 `+` 或 `≈` | 标记失败 |
| N4 | 口诀字数 | 是否 ≤ 15 字（考试应用 ≤ 30 字） | 标记失败 |
| N5 | 老师话术字数 | 话术是否约 500 字（允许 ±20% 浮动） | 标记失败 |
| 通用 | 字段完整性 | 所有必填字段非空 | 标记失败 |

> Layer 1 校验**仅做标记，不自动重新生成**。所有失败项汇总后进入人工审核环节（7.4），由人工决定是否触发重新生成。

### 7.3 Layer 2: AI 语义校验

**特点**：用另一个 Prompt 调用模型，对生成结果做语义层面检查。

校验项从 3.2 节原子化标准中"评估方式 = AI/人工判断"的规则派生：

| 对应规则 | 检查项 | AI 校验逻辑 |
|---------|--------|------------|
| M7 | 词义-词性匹配 | 释义是否与标注的词性一致 |
| C3 | 语块搭配合理性 | 是否为高频固定搭配，而非随意组合 |
| E1 | 例句语法难度 | 语法是否在初中教学大纲范围内 |
| E2 | 例句主干结构 | 是否为简单陈述句（主谓宾 / 主系表） |
| E3 | 例句连接词限制 | 是否仅使用 and/but/so/because |
| E4 | 例句从句限制 | 从句是否仅 that/which/who 引导，≤ 5 词 |
| E5 | 例句禁区检测 | 是否包含非谓语作状语、虚拟语气、倒装、独立主格 |
| E8 | 中文翻译语义对应 | 中文翻译是否与英文例句语义一致 |
| E9 | 例句义项匹配 | 例句中目标词的用法是否对应标注的义项 |
| E10 | 例句语言地道性 | 是否存在"中式英语"表达 |
| E11 | 例句内容安全性 | 是否涉及暴力、低俗、歧视、宗教政治等禁区内容 |
| N5 | 老师话术完整性 | 话术是否包含完整步骤框架 |
| N6 | 助记逻辑合理性 | 拆解逻辑是否通顺，是否为"伪助记" |

**注意事项**：

- AI 质检存在两类风险：**漏检**（放过不合格内容）和**过度检查**（误判合格内容为不合格）
- AI 质检不能替代人工审核
- AI 审核通过的内容仍需抽样人工复核

#### 7.3.1 质检 Prompt 组织策略

AI 语义校验的 Prompt 有两种组织方式，需在实现阶段根据实测结果选择：

**方案 A：分块检查（Per-rule Prompt）**

每条原子规则（如 C3 语块搭配合理性、E2 例句主干结构）使用独立的 Prompt 调用，一次只检查一个校验项。

| 维度 | 评价 |
|------|------|
| 精度 | 高——模型聚焦单一任务，判断更准确 |
| 可调性 | 高——某条规则漏检率过高时，可单独优化该 Prompt，不影响其他规则 |
| 版本管理 | 简单——每条规则独立迭代，独立记录漏检率/误判率 |
| API 成本 | 高——每条内容 × 每条规则各调用一次（如 13 条规则 × 2819 义项） |
| 延迟 | 高——调用次数多（可通过并发缓解） |

**方案 B：合并检查（Unified Prompt）**

将多条校验规则合并到一个 Prompt 中，模型一次性对所有维度做判断并返回结构化结果。

| 维度 | 评价 |
|------|------|
| 精度 | 中——规则间可能互相干扰，尤其规则数多时模型注意力分散 |
| 可调性 | 低——某条规则表现差时，调整 Prompt 可能连带影响其他规则 |
| 版本管理 | 复杂——改动牵一发动全身，需全量回归测试 |
| API 成本 | 低——每条内容只调用一次 |
| 延迟 | 低——调用次数少 |

**选择原则**：

- 规则数少、规则间关联度低 → 方案 B 更经济
- 规则数多、单条规则需要精细调优 → 方案 A 更可控
- 也可混合使用：将关联度高的规则合并为一组，不同组之间分块调用

> ⚠️ 具体方案待实现阶段在测试集上对比两种策略的漏检率、误判率和 API 成本后确定。

#### 7.3.2 质检 Prompt 的完整生产流程

质检 Prompt 本身也是一套 Prompt，必须走与内容生产 Prompt 相同的完整链路：

**Phase Q1: 质检 Prompt 编写**

- 背靠背写作：两人独立编写同一维度的质检 Prompt，对比合并
- 原子化：质检 Prompt 的每条判断规则必须拆解为可独立验证的原子检查项（即 7.3 表中的校验项）
- 版本管理：质检 Prompt 有独立的版本管理表，与内容生产 Prompt 分开管理

**Phase Q2: 质检模型选择**

- 质检模型可以与生成模型相同，也可以不同
- 用已知的"标准答案"（黄金测试集）+ "已知错误样本"作为测试集
  - **错误样本来源**：Phase 4 小批量验证中质检失败的真实案例 + 人工构造的典型错误样本（如语法超纲、伪助记、随意搭配等）
- 评估维度不是"通过率"，而是：

| 指标 | 定义 | 说明 |
|------|------|------|
| 漏检率（False Negative） | 不合格内容被放过的比例 | 越低越好，直接影响最终质量 |
| 误判率（False Positive） | 合格内容被误判为不合格的比例 | 越低越好，过高会增加不必要的返工 |

**Phase Q3: 定红线 + 优化质检 Prompt**

- 先定漏检率和误判率的红线（如：漏检率 ≤ 5%，误判率 ≤ 15%）
- 用测试集迭代优化质检 Prompt，直到达标
- 达标后质检 Prompt 进入 approved 状态

**质检 Prompt 版本管理表（独立于内容生产 Prompt）**：

| 字段 | 说明 |
|------|------|
| 版本号 | v1.0, v1.1 等 |
| 校验维度 | 对应 7.3 表中的哪些检查项 |
| 变更说明 | 修改内容和原因 |
| 变更日期 | YYYY-MM-DD |
| 对应模型 | 质检使用的模型 |
| 漏检率 | 测试集上的漏检率 |
| 误判率 | 测试集上的误判率 |
| 状态 | draft → testing → approved → deprecated |

> ⚠️ 质检 Prompt 的迭代需特别警惕**过度检查**问题：降低漏检率的同时容易推高误判率，两者需要平衡。

### 7.4 Layer 3: 人工审核

**审核范围**：

- Layer 1 算法校验标记为"失败"的项 → 必审（人工决定重新生成或手动修改）
- Layer 2 AI 质检标记为"不通过"的项 → 必审
- Layer 1 + Layer 2 均通过的项 → 抽样审核（比例待定）

**人工审核 Checklist**（对应 3.2 节中评估方式含"人工判断"或"人工/AI 判断"的规则）：

| 对应规则 | 审核要点 |
|---------|---------|
| M1 | 释义是否确实来源于权威词典，而非 AI 模糊生成 |
| M2 | 义项是否精炼到核心考点义项，非核心义项是否已过滤 |
| M7 | 词义与词性是否严格匹配，释义是否对应正确的词性 |
| P3 | 音标是否与权威词典一致 |
| C3 | 语块是否为高频固定搭配，而非随意组合 |
| E1 | 例句语法是否确实在初中教学大纲范围内 |
| E2 | 例句是否以简单陈述句结构为主 |
| E3 | 连接词是否仅使用 and/but/so/because |
| E4 | 定语从句是否仅用 that/which/who 引导，长度 ≤ 5 词 |
| E5 | 是否存在非谓语作状语、虚拟语气、倒装、独立主格等禁区结构 |
| E9 | 例句是否准确体现标注义项，是否存在张冠李戴 |
| E10 | 例句是否自然地道，是否存在"中式英语" |
| E11 | 例句内容是否阳光中性，无暴力低俗歧视等 |
| N5 | 老师话术是否包含完整步骤框架、逻辑是否通顺 |
| N6 | 助记逻辑是否合理，是否存在"伪助记" |
| N7 | 留空的助记项是否确实无法合理拆解（不是遗漏） |

**审核操作**：

| 操作 | 说明 |
|------|------|
| 通过 | 确认内容无误 |
| 重新生成 | 针对未通过的维度触发重新生成（不影响已通过的维度）。重新生成后需从 Layer 1 重新走质检流程。**每条内容最多重新生成 3 次**，超过 3 次后该操作不可用，必须转人工修改 |
| 人工修改 | 重新生成仍无法解决的问题（或已用完 3 次重新生成机会），人工直接修改。修改后需通过 Layer 1 算法校验（确保格式正确），无需再过 Layer 2 AI 校验 |

### 7.5 重新生成机制

```
质检完成（Layer 1 + Layer 2 标记结果）
    │
    ↓
人工审核（查看所有标记结果）
    │
    ├── 全部通过 → 审核通过 ✓
    │
    └── 存在失败项 →
            │
            ├── 人工点击"重新生成" →
            │       仅针对失败维度重新生成
            │       重新从 Layer 1 开始走质检流程
            │       （每条内容最多 3 次）
            │
            └── 已用完 3 次机会，或人工判断无需重试 →
                    人工直接修改
                    修改后过 Layer 1 校验确认格式正确
```

> **核心原则**：Layer 1 和 Layer 2 仅负责检测和标记，**不自动触发重新生成**。所有重新生成操作均由人工审核员在查看质检结果后主动触发。每条内容最多重新生成 **3 次**，超过后必须人工手动修改。

## 8. 维度扩展机制

**现有 5 个维度**：语义（Meaning）、语音（Phonetics）、音节（Syllables）、语块+例句（Chunk+Sentence）、助记（Mnemonic）

当未来需要新增内容维度时（如：近反义词辨析、词频分级等），按以下流程执行：

1. **标准制定**：学科专家输出新维度的学科标准
2. **内容生产 Prompt 编写**：走 Phase 1 完整流程（背靠背 → 原子化 → 版本管理）
3. **模型评估**：先为黄金测试集（50 词）补充新维度的标准答案，再用扩展后的测试集评估新维度 Prompt（可复用已选模型，也可重新选型）
4. **定红线 + 优化 + 小批量验证**：走 Phase 3–4 流程
5. **质检 Prompt 编写**：走 Phase Q1–Q3 流程（7.3.2 节），为新维度建立对应的 AI 质检能力
6. **量产 + 质检**：走 Phase 5–7 流程

> 新增维度不影响已有维度的内容。每个维度独立走完整流程。
