     │                                                                                                                                                                                                               │
     │ Context                                                                                                                                                                                                       │
     │                                                                                                                                                                                                               │
     │ 当前生产流程在 _generate_content 中一次性创建所有 asyncio Task 并 gather，有 1200s 全局超时。4000 词约产生 32,000-40,000 个 ContentItem，并发 20、每个 ~40s，需要 ~80,000s 才能跑完，远超 1200s 超时。Layer2  │
     │ 质检同理（虽然无超时，但一次创建数万个协程有内存压力）。                                                                                                                                                      │
     │                                                                                                                                                                                                               │

     │                                                                                                                                                                                                               │
     │ 方案：内部分批 gather                                                                                                                                                                                         │
     │                                                                                                                                                                                                               │
     │ 在 _generate_content 和 _collect_ai_results 内部将 items 按固定大小切片，每批独立 gather。改动集中在两个函数内部，外部接口零变更。                                                                            │
     │                                                                                                                                                                                                               │
     │ BATCH_SIZE = 200 的依据：200 items ÷ 20 并发 × 60s/item = 600s，远在 1200s 超时内。                                                                                                                           │
